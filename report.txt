 -Χρησιμοποιήθηκε το framework CAKEPHP
 -Χρησιμοποιήθηκε MYSQL βάση δεδομένων
 -Xρησιμοποιήθηκε ως Concurrent Versioning System το Git(Distributed CVS)


Λειτουργικότητα:

Πλήρης λειτουργικότητα
 -Έγγραφή χρήστη
 -Σύνδεση χρήστη
 -Επεξεργασία προφίλ
 -Αποσύνδεση χρήστη
 -Εμφάνιση αναφορών που έχει κάνει εγγεγραμένος χρήστης
 -Δημιουργία και κατάθεση αναφοράς
 -Πίνακας αναφορών του αναλυτή
 -Αξιολόγηση των αναφορών από τους αναλυτές

Τι λειτουργεί μερικώς:
 -Εμφάνιση δημοφιλών ειδών
 -Προσθήκη δημοφιλών ειδών
 -Επεξεργασία δημοφιλών ειδών
 -Διαγραφή δημοφιλών ειδών

Τι έχουμε να κάνουμε ακόμη:
 -Οτι αφορά τον υπεραναλυτή που δυστηχώς δεν προλάβαμε να το υλοποιήσουμε
 -Επιπλέον δυνατότητες του χρήστη(register validation μέσω email, αλλαγή κωδικού, διαγραφή λογαριασμού)
 -Πράγματα που αφορούν σε ασφάλεια(hash passwords, sanitize input)
    Σημείωση:Ήδη συμπεριλαμβάνουμε ελέγχους σχετικά με το input που δίνει ο χρήστης.

Unit Testing:

Ο προγραμματισμός που έχουμε κάνει ήταν ήδη test driven. Απλά ήταν χειροκίνητος
Για παράδειγμα στο register ενός χρήστη ελέγχαμε:
 Αν όταν έδινε λάθος είσοδο ο χρήστης το χειριζόταν σωστά το πρόγραμμα μας και δεν πέρναγε στη βάση η συγκεκριμένη εγγραφή.
Το άν πέρναγε στη βάση το κάναμε χειροκίνητα βλέποντας οι ίδιοι την βάση για να σιγουρέψουμε ότι σίγουρα δεν είχε εισαχθεί
μια εγγραφή που δεν θα έπρεπε. 
 Κάτι τέτοιο θα το κάνουμε στη δεύτερη φάση αυτόματα μέσω unit tests. Στη προκειμένη περίπτωση για παράδειγμα θα κάνουμε ένα 
test δίνοτας κατά το register, λάθος input. Εν συνεχεία, μέσα στο test, θα κάνουμε ένα ερώτημα στη βάση για να δούμε αν αυτή η εγγραφή έχει
εισαχθεί. Αν μας επιστρέψει ότι δεν έχει εισαχθεί, το test είναι επιτυχημένο. Αλλιώς το test θα θεωρείτε αποτυχημένο.
 Το ίδιο το framerwork που χρησιμοποιούμε έχει build in τρόπους για unit testing και το πιθανότερο είναι πως θα χρησιμοποιήσουμε
αυτούς. Σε αυτό το σημείο δεν έχουμε ασχοληθεί ακόμα με αυτό αλλά σίγουρα έχουμε στο μυαλό μας πως μπορεί να γίνει(με τον 
τρόπο που περιγράφεται μέσω του παραδείγματος παραπάνω).
